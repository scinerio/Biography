Experience

First Information Technology Services

Information Security Analyst

My role at FITS has been varying over time. The first few months were spent reviewing the vulnerability scan findings for Microsoft's entire Azure team (so many scans!). More specifically, I would write deviation requests that Azure would provide to FedRAMP to provide a clear indication of whether of not a scan finding is legitimate, scored properly, or unfixable by Azure (i.e. the vendor has not provided patches).

After this, I was the technical lead for a six month long technical privacy assessment of Microsoft Office products to identify any areas where personal data or the controls around it behaved unexpectedly. This project required knowledge of how to capture and review network traffic from iOS, Androd, Windows, and MacOS devices. Additional efforts were spent using automation tools (python and bash tools mostly) to sift through ~100 GB of telemetry data to ensure no personal data was colleted beyond the restrictions set in the Office Administration settings. The NDA prevents the findings from being shared, but Microsoft's engineering teams responded very quickly to any potential issues.

With my growing interest in penetration testing, I began self studying to meet the requirements to perform penetration tests on the FITS 3PAO team, acquiring the OSCP and GMOB certifications within about 6 months.

Since we don't have penetration tests queued up for me full time, I've been working as a Cloud Security Engineer for a FedRAMP cloud in Azure that we manage for Software AG Government. I've provided useful security alerts and dashboards to ensure that we have a high level of insight of potential threats to the environment. I'm also responsible for monthly vulnerability scanning and remediation, as well as providing guidance for cloud administration decisions from an offensive security perspective.

P.S. My explicit interest in becoming a security expert was obvious enough to FITS that I was promoted up two levels at once to match the skills I acquired in just one year.

Defense Point Security

Junior Cyber Defense Analyst

This position involved the use of numerous network monitoring tools to review and respond to various alerts for our clients. All of our clients had a Splunk instance that would be used to triage potential incidents, and other clients had additional security tools like RSA Netwitness and FireEye AX/HX/NX. A usual day at DPS could involve detecting and responding to DoS/DDoS attacks, determining if an endpoint has malware, or investigating ongoing phishing campaigns against our customers. Most notable alerts would require the creation of a report and actionable remediation tactics.
